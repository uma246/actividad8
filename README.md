PREGUNTAS:
1.Sirve para indicarle a Git qué archivos o carpetas no debe seguir ni incluir en el repositorio para asi evitar subir archivos innecesarios.

2.Es conveniente ignorar archivos temporales, archivos de errores (logs), configuraciones locales y archivos que se generan automáticamente, ya que no son necesarios para que funcione el proyecto en otros equipos.

3. Los archivos ignorados no se suben a GitHub porque Git no los sigue, pero el archivo .gitignore sí se sube porque contiene las reglas (*.log, *.tmp, *.local).
